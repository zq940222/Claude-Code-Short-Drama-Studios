# /edit 粗剪预览与精剪交付包（可选）

> **定位**：粗剪只是"快速看节奏"的预览，不是成片。正式成片请走 `/finalcut`（剪映/达芬奇时间线，
> 非破坏性、质量最好）。本命令的核心价值是**整理精剪交付包**（原始素材 + 精剪指导书）；
> 粗剪预览可做可不做。不想看预览、直接精剪的话，跳过本命令用 `/finalcut` 即可。

## 运行时适配（跨 Agent 兼容）

- **支持 subagent 的运行时**（Claude Code / Hermes Agent 等）：按下文"调度 xxx agent"正常派发子代理执行
- **不支持 subagent 的运行时**（OpenClaw 等，以插件 bundle 方式安装）：正文提到"调度 X agent"时，
  改为读取插件根 `agents/<X>.md`（本技能目录上两级），以其为工作规范在当前上下文直接执行，效果等同
- **用户确认处**：有 AskUserQuestion 工具就用；没有则直接在对话中提问并等待用户回复，门禁语义不变


## 前置检查

- shotlist.json 中本集全部镜头 status=success（或用户在 /review 明确放行）；有缺口先报告

## 流程

0. 工作区缺 `tools/concat.py` 时，先从插件根（本技能目录上两级）的 `tools/concat.py` 复制过来
1. 调度 **editor** agent：
   - **精剪交付包（核心，必出）**：`05-final/delivery-ep{NN}.md`——原始素材清单（sh*.mp4/BGM/SRT/片头尾）
     + EDL 式剪辑顺序表 + 手工精剪步骤（原始 sh*.mp4 是唯一真源，只读不改）
   - **粗剪预览（可选，用户想看节奏才做）**：用 `tools/concat.py`（默认无损流拷贝，规格不一致才转码）
     拼接，按 `editing` 块处理增强选项（默认全关），输出 `05-final/<剧名>-ep{NN}-粗剪.mp4`
2. 向用户报告交付包路径；做了粗剪则一并报告预览路径并展示抽帧
3. 收尾提示：**正式成片走 `/finalcut`（AI 自动组装剪映/达芬奇时间线）**；也可拿交付包在剪映/PR 手工精剪
   （步骤见 delivery-ep{NN}.md）。多集项目提示继续下一集
